# Zerno Cafe

Веб-приложение кофейни **Zerno**: заказ еды на вынос, бронирование столов и панели для сотрудника и администратора.

## Возможности

### Клиент
- Каталог меню, корзина, оформление заказа с выбором даты и времени получения
- **Бронирование столов** — интерактивная схема зала:
  - свободные и занятые столы для выбранного времени;
  - подсказка при наведении: когда стол освободится;
  - клик по занятому столу — переход времени начала к моменту окончания брони;
- Профиль: заказы и **бронирования** (предстоящие / история, отмена ожидающих броней)
- Email-уведомления о статусе заказа и коде выдачи

### Сотрудник
- Обработка заказов и смена статусов
- Брони столов: список, подтверждение, завершение, отмена
- Схема зала в реальном времени (кто сейчас за столом)

### Администратор
- Меню, категории, сотрудники, отчёты
- Редактор схемы зала (столы, маркеры: вход, стена, касса)

## Запуск

```bash
npm install
npm run setup:env   # Windows: создать .env из примера
npm start           # http://localhost:3000
```

Для разработки с автоперезагрузкой: `npm run dev`.

### Переменные окружения (`.env`)

| Переменная | Описание |
|------------|----------|
| `JWT_SECRET` | Секрет для токенов |
| `SMTP_*` | Почта для кодов и уведомлений (опционально) |

## API (бронирование)

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/tables` | Список столов (схема) |
| `GET` | `/api/floor-markers` | Маркеры зала |
| `GET` | `/api/tables/availability?date&time&duration_minutes&guests?` | Занятость на слот + `conflict` (время окончания блокирующей брони) |
| `POST` | `/api/bookings` | Создать бронь (JWT клиента) |
| `GET` | `/api/my-bookings` | Мои бронирования |
| `PUT` | `/api/bookings/:id/cancel` | Отмена (только статус `ожидает`) |

Статусы брони: `ожидает` → `подтверждено` → `завершено` / `отменено`. На схеме слот занят только после `подтверждено`; если гость не пришёл в течение 30 мин от начала — `отменено`, стол свободен.

Правила времени (МСК): работа 10:00–22:00, бронь с шагом 30 мин, минимум 1 ч, не позже 21:30 на начало, +30 мин от «сейчас» на сегодня.

## Структура

```
backend/          Express + SQLite
frontend/client/  Сайт клиента
frontend/employee/ Панель сотрудника
frontend/admin/   Панель администратора
frontend/shared/  Общие стили, схема зала, утилиты
```

## Технологии

Node.js, Express, SQLite, JWT, vanilla JS/HTML/CSS.
